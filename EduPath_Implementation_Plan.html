EduPath

Complete Implementation Plan
From Demo → Production College Mentoring Platform
Version 1.0 · May 2026
Important
This is a large-scale transformation from a demo landing page into a production college mentoring app. Organized into 4 phases that ship incrementally. Each phase builds on the last and is independently deployable.

Confirmed Decisions

🏷️ Branding

Keep as "EduPath" — tagline: "EduPath — Intelligent College Mentoring"

🏛️ College Structure

B.Tech — 4 years, 8 semesters (2 per year)

8 Departments:

  1. Computer Science (CS)
  2. Computer Science — AIML
  3. Computer Science — Cyber Security
  4. Computer Science — Data Science
  5. Electronics & Communication Engineering (ECE)
  6. Electrical & Electronics Engineering (EEE)
  7. Civil Engineering
  8. Mechanical Engineering

ECE syllabus will be provided to seed the first curriculum.

👥 Role System

Current: admin | teacher | student
New: admin | teacher | student | parent
Parents linked to students via a parent_student_links table

📅 Semester-Based Data

All academic data scoped to a semester entity with start_date, end_date, is_active. Enables the "Time Machine" archival feature.

💰 No Paid APIs

  • AI mentoring: manual transcript paste (no Whisper)
  • AI chat: existing Supabase Edge Function (free tier)
  • No external paid services required

🔔 Notifications

  • Phase 1: In-app notifications only (free)
  • Phase 2: Email via Resend free tier (100 emails/day)
  • Future: WhatsApp Business API (deferred — requires paid plan)

📝 Data Entry

No existing student data — build Admin UI for manual entry.

📚 Notes System

Rich-text markdown stored per-topic. Teachers create/edit; students view. Powers the "Click-to-Notes" knowledge graph feature.

PHASE 2 Teacher & Mentoring Experience

Estimated: ~3-4 days

Important
Mentoring is the core identity of EduPath. This phase comes before student features because the mentoring infrastructure defines the student experience.

2.1 Mentor-Student Assignment System

  • Admin assigns mentors to students via /admin/mentors
  • Each teacher sees assigned mentees on /teacher/mentees
  • Mentee Card: Photo, name, department, semester, risk score, last pulse check, last session date
  • Teachers can view any mentee's full profile: graph, attendance, assignments, CGPA

2.2 AI Mentoring Suite

Note
No paid APIs. Session notes are manual text entry/paste. AI summarization uses the existing free Supabase Edge Function.

New Tables

TablePurpose
mentoring_sessionsmentor_id, student_id, scheduled_at, status, transcript, ai_summary, action_items
mentoring_notessession_id, note_text, created_by

UI Components

  • Pre-Session Briefing Card: AI generates a briefing from recent mastery changes, missed assignments, attendance dips, prior session notes, pulse check trends
  • Session Notes Entry: Rich text area for typing/pasting session notes
  • Post-Session AI Summary: AI analyzes notes → generates summary + action items (saved to DB)
  • Session History Timeline: All past sessions with a student, searchable

2.3 Teacher Dashboard — Class Intelligence

  • Risk-Sorted Roster: All students sorted by risk score, with sparklines
  • Pulse Check Overview: See all students' latest pulse ratings (🔴🟡🟢)
  • Subject Mastery Heatmap: Grid of student × topic, color-coded
  • Assignment Manager: Create/edit, view submissions, grade in bulk
  • Announcements: Post per-department or per-semester notices

2.4 Context-Aware Intervention Plans

When risk score > threshold OR pulse check drops below 3:

  • System auto-generates a concrete intervention plan
  • Specific topics to review (from topic notes)
  • Suggested study schedule for the next 2 weeks
  • Recommended resources
  • Teacher can approve/edit/send as in-app notification

2.5 Predictor (Absorbed)

The standalone predictor page becomes a tool inside /teacher/graph. Teachers can run "what-if" scenarios on any student's data.

PHASE 3 Student Experience & Click-to-Notes

Estimated: ~3-4 days

3.1 Knowledge Graph — Click-to-Notes

Clicking a node opens a slide-out panel with:

  • Topic name, difficulty, mastery arc
  • 📝 Study Notes tab — rendered markdown
  • 📊 My Progress tab — mastery history chart
  • 🔗 Prerequisites — clickable links to prereq nodes
  • 🤖 Ask AI — context-aware question input

Teachers see an "Edit Notes" button that opens a markdown editor.

3.2 Proactive AI Study Path

AI analyzes the knowledge graph topology and suggests:

  • "Study Hash Tables next — it unlocks 4 downstream topics"
  • "Your weakest prereq chain is: Logic → Sets → Combinatorics. Start with Logic."

Shown as a "What to Study Next" card on the dashboard. Uses graph traversal + mastery data (runs client-side, no API needed).

3.3 Dynamic Semester System

  • Semester picker in the header
  • Knowledge graph, subjects, mastery all filter by selected semester
  • Time Machine toggle: Switch to past semesters → frozen mastery data

3.4 Weekly Pulse Check

Simple widget: "How's studying going this week?"

5 emoji options: 😫 😟 😐 🙂 😄

Optional short note. Mentor sees aggregated pulse data on their dashboard.

3.5 Assignment & Task Hub

  • All pending assignments, sorted by due date
  • Status badges: Pending / Submitted / Overdue / Graded
  • Grouped by subject with progress bars
  • Quick-submit button (mark done + optional file link)

3.6 Attendance Dashboard

  • Calendar heatmap showing present/absent/late days
  • Subject-wise percentages with threshold warnings (< 75%)
  • Trend line showing trajectory

3.7 Announcements Feed

Students see relevant announcements filtered by department + semester. Pinned ones stay at top.

3.8 AI Study Aid Generator

  • On any topic note: "Generate Quiz" / "Generate Flashcards" / "Summarize"
  • Uses AI edge function with note content as context
  • Interactive flashcard UI with flip animations

PHASE 4 Admin, Parent & Advanced Features

Estimated: ~3-4 days

4.1 Admin Dashboard

  • KPI Cards: Total students, avg CGPA, avg attendance, high-risk count
  • Semester Manager: Create semesters, assign subjects, bulk import attendance/CGPA
  • User Manager: View all users, assign roles, link parents to students
  • Resource Allocation Heatmap: Topics where >40% of students scored <50% mastery

4.2 Parent Dashboard

  • Child Progress: CGPA trend, attendance %, risk score, upcoming deadlines
  • Weekly Learning Stories: Auto-generated summary via email (Resend free tier)
  • Example: "This week, Aarav mastered 3 topics in Data Structures but missed 2 classes in OS. Risk score: 45 → 38."
  • Read-only access: Knowledge graph, attendance calendar, assignment status

4.3 Career Readiness Checklist (Simplified)

  • Simple checklist per career track (e.g., "Data Scientist")
  • Shows: required topics (✅/❌), minimum CGPA (✅/❌), skills needed
  • Clean progress card — no complex graph overlay

4.4 Coding Log (Simplified)

  • Simple manual daily log: "I solved 3 LeetCode problems today"
  • Dashboard shows streak counter + weekly chart
  • No API integrations (kept simple and free)

4.5 Extracurricular Profile (Simplified)

  • Simple list on student profile: event name, date, type
  • Portfolio section — no complex tracking
